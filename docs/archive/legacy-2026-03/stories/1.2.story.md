### Argument Parsing Requirements

**参数解析函数** [Source: docs/prd.md Section 2.1 - FR3]:

**输入**: 字符串，例如：
- `"--yolo"`
- `"--allow-all-tools --verbose"`
- `"--flag value --another 'quoted value'"`
- `"--path=\"/path/with spaces/\""`

**输出**: 字符串数组，例如：
- `["--yolo"]`
- `["--allow-all-tools", "--verbose"]`
- `["--flag", "value", "--another", "quoted value"]`
- `["--path=/path/with spaces/"]`

**解析规则**:
1. **空格分隔**: 默认按空格分割参数
2. **引号处理**: 单引号/双引号内的空格不分割
3. **转义字符**: 支持 `\` 转义引号
4. **空字符串**: 返回空数组 `[]`
5. **错误处理**: 未匹配的引号应记录警告但不阻塞

**推荐实现** [Source: Node.js/Shell标准实践]:
```typescript
/**
 * Parse command-line arguments string into array.
 * Handles quotes, spaces, and escape characters.
 * 
 * @param argsString - Raw arguments string from configuration
 * @returns Array of individual arguments
 */
function parseArgumentsString(argsString: string): string[] {
  if (!argsString || argsString.trim() === '') {
    return [];
  }
  
  const args: string[] = [];
  let current = '';
  let inQuote: string | null = null;
  let escaped = false;
  
  for (let i = 0; i < argsString.length; i++) {
    const char = argsString[i];
    
    if (escaped) {
      current += char;
      escaped = false;
      continue;
    }
    
    if (char === '\\') {
      escaped = true;
      continue;
    }
    
    if (char === '"' || char === "'") {
      if (inQuote === char) {
        inQuote = null;  // 关闭引号
      } else if (inQuote === null) {
        inQuote = char;  // 开启引号
      } else {
        current += char;  // 引号内的其他引号字符
      }
      continue;
    }
    
    if (char === ' ' && inQuote === null) {
      if (current.length > 0) {
        args.push(current);
        current = '';
      }
      continue;
    }
    
    current += char;
  }
  
  if (current.length > 0) {
    args.push(current);
  }
  
  // 警告未匹配的引号
  if (inQuote !== null) {
    log.warn(`[Codebuddy] Unmatched quote in arguments: ${argsString}`);
  }
  
  return args;
}
```

### Error Handling Strategy

**错误场景** [Source: docs/prd.md Section 2.1 - FR5]:
1. **未匹配的引号**: 记录警告，继续解析
2. **过长参数**: 参数总长度 > 1000 字符 → 警告用户
3. **特殊字符**: 不阻塞，但记录日志
4. **配置读取失败**: 使用空数组后备

**用户反馈机制**:
```typescript
// 示例：参数过长警告
if (customArgsString.length > 1000) {
  vscode.window.showWarningMessage(
    `Custom arguments for ${currentToolId} are very long (${customArgsString.length} chars). This may cause issues.`
  );
}

// 示例：未匹配引号警告
if (inQuote !== null) {
  vscode.window.showWarningMessage(
    `Arguments for ${currentToolId} have unmatched quotes. Parsing may be incorrect.`
  );
}
```

2. **VS Code API 限制**: `shellArgs` 只在终端创建时生效，无法动态修改已存在终端的参数
3. **日志调试**: 使用 `log.debug()` 输出原始参数字符串和解析后数组，便于调试
4. **终端复用**: 现有终端复用逻辑会跳过参数重新应用，这是预期行为（用户需关闭终端才能应用新参数）

### Known Limitations

- **终端复用限制**: 已存在的终端不会重新应用参数更改，用户需关闭终端重新启动
- **平台差异**: 不同操作系统的 shell 对参数处理可能略有不同，重点测试 macOS/Linux（zsh/bash）
- **参数验证**: 不验证参数是否对特定工具有效（由工具本身验证）

### Security Considerations

- **命令注入风险**: `parseArgumentsString` 不执行代码，仅解析字符串，风险较低
- **用户配置信任**: 假设用户配置是可信的，不过滤危险参数（用户负责）
- **日志敏感信息**: 参数可能包含敏感路径或配置，使用 debug 级别日志
